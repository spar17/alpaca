# ALPACA: Automated Pipeline for Carnatic Music Analysis

Code accompanying the paper **"Musically-Constrained Circular Mixture Models for Swaram Inventory Estimation in Carnatic Music"** (ISMIR 2026, under review).

---

## Repository Structure

```
alpaca/
├── notebooks/
│   ├── alpaca.ipynb          # Main analysis: VMM-C methodology and evaluation
│   └── preprocessing.ipynb  # Audio preprocessing and feature extraction
├── figures/                  # All figures generated by the analysis
├── outputs/
│   └── annotations/          # Generated Sonic Visualiser annotation files (.svl)
└── tables/                   # Generated LaTeX table snippets
```

## Dataset

This repository does not include audio data. The experiments use the publicly available **Carnatic Varnam Dataset**:

> Koduri, G. K., et al. *Carnatic Varnam Dataset.* Freesound, 2013.  
> Download: [https://freesound.org/people/gopalkoduri/packs/13116/](https://freesound.org/people/gopalkoduri/packs/13116/)

After downloading, place the dataset at `carnatic_varnam_1.0/` in the repo root. The expected layout is:

```
carnatic_varnam_1.0/
├── Audio/                        # .mp3 recordings
└── Notations_Annotations/
    └── annotations/
        └── taalas/               # Per-ragam, per-artist .svl annotation files
```

## Notebooks

**`notebooks/alpaca.ipynb`** is the primary notebook. It is organized as follows:

| Section | Content |
|---|---|
| 0. Preprocessing | pYIN pitch extraction, contour smoothing, annotation parsing |
| 1. Understanding the Swara | Segment analysis, gamakam statistics, anchor-pair effects |
| 2. Tonic and Label Estimation | VMM-C methodology, musically-informed constraints, evaluation ← *paper content* |
| 3. Contour Segmentation | Beat-aligned segmentation evaluation |
| 4. Tempo Estimation | Akshara distribution, syllable detection |
| 5. Dynamic Programming | Full end-to-end transcription pipeline |

For the paper's core contribution, focus on **Section 2**.

**`notebooks/preprocessing.ipynb`** covers standalone preprocessing steps before the main analysis.

## Outputs

`outputs/annotations/` contains generated Sonic Visualiser (`.svl`) annotation files for each recording in the dataset. These can be loaded alongside the audio in [Sonic Visualiser](https://www.sonicvisualiser.org/) for qualitative inspection.
